# Проект #2: Анализ текстовых данных на Flask

Проект является логическим продолжением предыдущего и посвящен созданию простейшего веб-приложения на микрофреймворке `Flask`.

Подразумевается, что будет переиспользована логика (юз-кейсы) из предыдущего проекта, а использование `Flask` ограничится 
реализацией тонкого слоя взаимодействия с пользователем, альтернативы консольному режиму из предыдущего проекта.*

Контрольная точка предполагает два варианта: I для номеров с 1 по 12 по списку, II вариант для последующих.

Оба варианта делают функционал статистики. **I вариант** помимо нее реализует топ-слова, **II вариант** - облако слов. 

## Сценарии использования

#### Функционал "Статистика"

| Номер | Действие | Результат |
|---|---|---|
| 1 | Пользователь переходит по корневому адресу | Отображается страница с формой для выбора файла |
| 2 | Пользователь выбирает файл для обработки | Проверка корректности файла (на клиенте) |
| 2.A | ...выбранный файл - корректный | Отображение элементов интерфейса для выбора действия |
| 2.B | ...выбранный файл - некорректный | Сообщение-подсказка об ошибке, возврат к пункту `2` |
| 3 | Пользователь выбирает желаемое действие и нажимает кнопку | Отправка формы, обработка формы |
| 3.A | ...действие - "Статистика" | Перенаправление на страницу `/stats`, отображение статистики в табличном виде |
| 3.B | ...действие - "Топ-слова" | Перенаправление на страницу `/top`, отображение топ-слов в табличном виде и изображения с облаком слов |
| 3.C | ...не удалось обработать файл | Сообщение-подсказка об ошибке, возврат к пункту `2` |

## Критерии оценивания

**Функционал**: 

- стартовая страница: до двух баллов
- функционал статистики: до двух баллов
- функционал топ-слов: до четырех баллов

**Штрафы**:
- минус балл при несоблюдении норм форматирования и наличии ошибок линтера
- минус балл за несоответствие рекомендованной структуре

## Настройка окружения и запуск

### Активация виртуального окружения

**Windows**

```sh
venv\Scripts\activate
```

**Mac / Unix**

```sh
source venv\bin\activate
```

### Билд / установка пакета

```sh
pip install -e .
```

### Запуск приложения

```sh
web
```

### Работа в системе контроля версий

Создайте ветку со своей фамилией

```sh
git switch -c фамилия-измени-меня
```

Далее фиксируйте изменения по мере продвижения

```sh
git add .
git commit -m "работа над заданием"
```

Если гит выдает ошибку, почитайте и погуглите её: могут быть не настроены имя и почта, может отсутствовать сам гит.

Если всё ок, залейте вашу ветку на гитхаб:

```
git push -u origin фамилия-измени-меня
```

Далее можете делать (или не делать) промежуточные коммиты.

**Главное, не забудьте закоммитить и запушить финальные изменения!**
